# Q3 2026 Brag

## Impact

**Cracked the 12-day CoinDCX execution blocker (2026-07-01):** After 12 days of diagnosing a generic `400 Invalid request` on every order, isolated root cause as `min_notional` — the client validated `min_quantity` but not minimum order value ($5 on BTCUSDT). Quantity-valid, value-too-small orders were silently passing local validation and hitting CoinDCX's server. A reference-library live test proved the account was never blocked. Fix: `BelowMinNotionalError` raised locally before any network call; 21/21 tests pass. System unblocked for live capital. See [[CoinDCX Execution Layer]].

**Built and panel-reviewed Phase 2 live execution — automated SL/TP bracket (2026-07-01):** `place_sl_tp_orders()` places `stop_limit` SL and `take_profit` TP immediately after entry. Before shipping, ran `/panel` adversarial review (0 proceed / 2 revise / 1 kill), which caught and forced fixes on three issues: orphaned SL when TP placement fails, wrong order type (`take_profit_limit` doesn't exist on CoinDCX — caught live), and a kill-level exit detection flaw (yfinance blind to intraday fills). Rewrote monitor to poll CoinDCX order status first. Tests: 31 → 38, all passing. See [[CoinDCX Execution Layer]].

## Technical Growth

**"Test the API, don't read the docs":** Two session catches came from live tests, not documentation: `take_profit_limit` not a valid CoinDCX type (only found by placing the order); the 400 root cause exposed by running a reference library and getting HTTP 200. Docs are a hypothesis. The API response is the ground truth.

**Panel-before-push discipline on money-path code:** `/panel` ran before committing Phase 2. Fixed all panel findings in the same session before the first push. Pattern documented in [[Patterns#Panel → fix loop before shipping money-path code]].

**Built Weekly Hunt cross-instrument harness (2026-07-02):** `hunt/cross_instrument_runner.py` — gauntlet step 3. Runs any `backtest_fn(df) -> {wins, trades}` on train slices of BTC, XAU, EUR; requires unanimous 3/3 pass (luck doesn't replicate across uncorrelated asset classes). Key design: exceptions per-instrument count as fail without aborting the run; n_trials captured from registry at call time (so the Bonferroni-raised bar is consistent). 12 tests green including lockbox isolation proof (spy confirms backtest_fn never sees data >= 2024-01-01). The Weekly Hunt gauntlet is now runnable end-to-end. See [[Daily Watch Weekly Hunt]].

**Added BTC regime to Daily Watch brief (2026-07-02):** `brief.py` now surfaces BTC price, EMA50/EMA200 regime status, and 20-bar high proximity in every morning Telegram brief. Golden-cross detection (the go-live trigger) is now part of the daily read. See [[Trading System]].
